# rebuild-mapping

本文档将记载auto-cloud-gi-v2版本全流程中涉及到的所有图形界面以及点击/等待的需求，并将通过文字描述+图片的形式展示（部分图片为16：10，我们的项目基本运行在16：9上，但即使这样我也希望程序与比例无关）

## 云原神每日初次登录领取免费时长界面

![050213.571](https://luweiphoto.oss-cn-wuhan-lr.aliyuncs.com/202604140924709.png)

点击空白区域就可以关闭，一般来说是点击下方的“点击空白区域关闭”就可以关闭

## 云原神初始界面

![image-20260414092400894](https://luweiphoto.oss-cn-wuhan-lr.aliyuncs.com/202604140924519.png)

比较重要的按钮就是右下角的“开始游戏”

## 点击开始游戏后的选择界面

点击“开始游戏”后有这样几个分支：

一个是畅玩卡，则会直接进入排队/等待进入游戏

一个是非畅玩卡玩家，这就会弹出弹窗，要玩家选择使用免费/付费时长去加载进入游戏。

## 非畅玩卡玩家路径

畅玩卡玩家不会执行到这里。

非畅玩卡路径先留空，我认为应该通过图像识别检测来判断是哪种途径（过程处于哪个状态），后续再补全剩余的一些逻辑，这部分对应了v2版本里的快速队列/免费队列/原点队列（付费）。

## 原神空月女神界面

要双击获得原石，可能还会需要再单击以退出原石界面。

![050414.526](https://luweiphoto.oss-cn-wuhan-lr.aliyuncs.com/202604140947212.png)

下侧有字样“点击领取今日空月祝福奖励”，图案会随账号不同而不同。

双击字样处就会领取，随后获取原石的界面再单击就OK了。

这个东西让我对咱们的流程控制有一些思考吧，这个行为可以封装为原来的那种点击+sleep+点击去兜底，但是也可以拆开，所以我在想项目的流程控制需要的粒度是怎样的呢？以及有没有通用一点的异常处理，识别到界面怎样了进行怎样的处理。
